# Базовые сценарии использования (Happy Paths)

---

## 1. Онбординг и начисление стартового баланса
**Цель:** Пользователь регистрируется и получает виртуальные деньги для старта.

* **Предусловия:** Пользователь не авторизован в системе.
* **Шаги:**
  1. Пользователь открывает PWA и нажимает "Войти/Зарегистрироваться" (например, через OAuth-провайдера).
  2. `Auth Service` создает профиль в БД и инициализирует кошелек.
  3. `Wallet Service` начисляет стартовый бонус (например, 1000 монет) и записывает это в Ledger.
  4. Frontend получает JWT-токен и отображает актуальный баланс в шапке.
* **Постусловие:** Пользователь авторизован, на балансе начислены стартовые монеты.

---

## 2. Просмотр рынков и котировок
**Цель:** Пользователь изучает доступные события и текущие шансы.

* **Предусловия:** Пользователь авторизован (или является гостем, если просмотр разрешен).
* **Шаги:**
  1. Пользователь открывает главную страницу (ленту рынков).
  2. Frontend запрашивает список активных рынков у `Market Service`.
  3. `Market Service` отдает данные из Redis (кэш), включая заголовки и текущие проценты/коэффициенты.
  4. Пользователь кликает на конкретный рынок (например, "Выборы в США").
  5. Frontend запрашивает детали: описание, объем пулов по каждому исходу, график изменения odds.
* **Постусловие:** Пользователь видит актуальную информацию и готов сделать ставку.

---

## 3. Размещение ставки (Критический путь)
**Цель:** Пользователь успешно ставит виртуальные деньги на исход.

* **Предусловия:** У пользователя достаточно средств, рынок находится в статусе `OPEN`.
* **Шаги:**
  1. Пользователь выбирает исход (например, "Да") и вводит сумму (например, 100 монет).
  2. Frontend показывает потенциальный выигрыш на основе текущих odds.
  3. Пользователь нажимает "Подтвердить ставку".
  4. `Betting Engine` проверяет статус рынка и баланс пользователя.
  5. Система резервирует/списывает 100 монет с баланса.
  6. Система добавляет 100 монет в общий пул исхода "Да".
  7. Система пересчитывает odds для этого рынка (с учетом 5% комиссии платформы) и рассылает обновления по WebSocket/SSE другим пользователям.
  8. Frontend показывает экран успеха и обновленный баланс.
* **Постусловие:** Ставка создана, баланс уменьшен, пул рынка увеличен, odds изменились.

---

## 4. Разрешение рынка и выплаты (Системный процесс)
**Цель:** Система корректно закрывает рынок и распределяет выигрыши.

* **Предусловия:** Событие в реальном мире произошло, Scraper получил результат.
* **Шаги:**
  1. `Scraper Worker` видит в API Polymarket, что рынок перешел в статус `RESOLVED` с выигрышным исходом "Да".
  2. Scraper отправляет событие `MarketResolved` в Kafka.
  3. `Market Service` блокирует рынок для новых ставок (статус `CLOSED`).
  4. `Wallet Service` находит все выигрышные ставки по этому рынку.
  5. Система забирает 5% (rake) в пользу платформы, а оставшиеся 95% распределяет между победителями пропорционально их ставкам.
  6. Средства зачисляются на балансы победителей, записи фиксируются в Ledger.
  7. Пользователям-победителям отправляется Push-уведомление.
* **Постусловие:** Рынок закрыт, победители получили выплаты, платформа забрала комиссию.

---

## 5. Монетизация: Отключение рекламы
**Цель:** Пользователь тратит виртуальную валюту на улучшение UX.

* **Предусловия:** У пользователя есть свободный баланс, он видит рекламу.
* **Шаги:**
  1. Пользователь заходит в настройки профиля и видит опцию "Отключить рекламу на 24 часа за 500 монет".
  2. Пользователь нажимает "Купить".
  3. `Monetization Service` проверяет баланс и списывает 500 монет.
  4. В профиле пользователя устанавливается флаг `ad_free_until = now() + 24h`.
  5. Frontend скрывает рекламные блоки.
* **Постусловие:** Баланс уменьшен, реклама скрыта, валюта "сожжена" (сток).

---

## 6. Банкротство и перезапуск
**Цель:** Пользователь, потерявший все деньги, может начать заново.

* **Предусловия:** Баланс пользователя = 0, нет активных (незакрытых) ставок.
* **Шаги:**
  1. Пользователь заходит в приложение, видит баланс 0 и баннер "Вы обанкротились".
  2. Пользователь нажимает "Начать заново".
  3. `Wallet Service` проверяет отсутствие активных ставок и начисляет стартовый бонус (например, 500 монет — меньше, чем при первой регистрации).
  4. В Ledger записывается транзакция типа `BANKRUPTCY_RESET`.
* **Постусловие:** Баланс восстановлен, история банкротств сохранена для аналитики.
